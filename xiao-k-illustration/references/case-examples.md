# Case Examples

这些案例展示推理过程，不是固定模板。

## AI tool introduction

正文观点：一个工具不只会聊天，还能调用多个工作流程。

- 结构：系统局部。
- 小K职责：打开工具箱并接通一个入口。
- 物件：工具箱、三条任务轨道。
- 画面：小K掀开工具箱，里面不是工具图标，而是文档、表格和浏览器三条轨道开始运行。

## Step-by-step tutorial

正文观点：先给参考，再拆结构，最后让 AI 重构页面。

- 结构：路线。
- 小K职责：带着同一张参考卡依次通过三个工作台。
- 物件：参考卡、三个简化工作台。
- 画面：小K用弹簧腿跨过“参考、拆解、重构”三个动作节点，最终得到自己的页面窗口。

## AI safety warning

正文观点：给 AI 全部权限之前必须设置确认点。

- 结构：概念隐喻。
- 小K职责：在文件进入粉碎入口前踩住刹车。
- 物件：文件轨道、黄色确认闸门。
- 画面：尾钟快速转动，小K用打结的弹簧腿踩住闸门，提醒先确认再执行。

## Knowledge system

正文观点：资料存下来不等于以后能找回来。

- 结构：前后对比。
- 小K职责：左侧在杂乱窗口里翻找，右侧从带标签的档案抽屉准确取回同一张卡。
- 物件：窗口堆、档案抽屉。

## Web coding

正文观点：AI 做网页时，先给真实设计参考，比说“高级一点”更有效。

- 结构：对比。
- 小K职责：左侧对着空白窗口抓头，右侧把参考框架铺在窗口上逐层替换内容。
- 物件：空白浏览器、参考框架。

## One idea, multiple platforms

正文观点：一条素材可以重构成公众号、小红书和短视频，但不能简单复制。

- 结构：分流。
- 小K职责：把一个原始内容包分别装进三种不同形状的发布容器。
- 物件：内容包、三个不同平台容器。
- 重点：三个结果保留同一核心，但尺寸和内部结构不同。

## Open-source Xiao-K article

正文分段与建议配图：

1. 角色为什么需要固定视觉语言
   - 小K站在四面风格完全不同的画框中间，努力把它们拉回同一条轨道。
2. 一张参考图为什么还不够
   - 小K拿着校准尺检查自己的角、耳机、尾钟和弹簧腿，旁边是几个漂移的错误零件。
3. Skill 能做什么
   - 小K操作一个内容工作台，同一份内容被分别重构成横向文章图、竖向图文页和视频章节卡。
4. 为什么开源
   - 小K打开自己的工具箱，把可复用的角色规则和构图卡递给画面外的人，而不是复制角色本身。

这四张图都不是单纯表情图。小K分别承担拉回、校准、重构和分享四种动作。
