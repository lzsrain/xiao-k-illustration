# Xiao-K Scene Grammar

## Action library

根据文章核心选择一个动作，不为了热闹堆动作。

- 连接：插、接、拧、扣、搭桥。
- 整理：分、叠、装箱、标记、归档。
- 实操：点、拖、输入、测试、对比。
- 修复：补、扶、拉回、踩停、拆除。
- 生产：加工、打包、分发、发布、回收。
- 探索：探路、照亮、试错、绕行、测量。
- 承压：顶住、被压、陷入、拖行、追赶。

## Contact rule

每张场景图至少出现一个清楚的物理接触点：

- 手接触按钮、线缆、工具或内容物。
- 脚踩住刹车、踏板、台阶或断点。
- 身体顶住、卡进、拖动或承受某个结构。
- 尾钟勾住时间线、触发节点或指向结果。

小K只是站在旁边看、指着物件或做表情，不算参与。没有接触点就重新设计。

## Role library

- 操作员：进入机器、控制开关、调整参数。
- 搬运工：扛、拖、装箱、分发内容。
- 维修工：补缺口、接断线、扶正结构。
- 测试员：对比两个结果、测量、检查、复盘。
- 守门员：踩刹车、确认权限、拦住错误输出。
- 分享者：打开工具箱，把可复用的方法递出去。

## Body acting

让小K的身体参与叙事：

- 弹簧腿压缩：压力、卡住、准备启动。
- 弹簧腿拉长：跨越、赶路、突然提速。
- 弹簧腿打结：翻车、流程冲突、信息过载。
- 身体前倾：推进、执行、赶时间。
- 身体塌下：疲惫、失败、负担太重。
- 尾钟加速：截止时间、热点、紧急任务。
- 尾钟停住：卡点、等待确认。
- 尾钟指向完成：结果验证、流程跑通。

## Object library

只选与内容有关的 1-2 件。

### AI and tools

- 提示词卡、模型插槽、浏览器窗口、权限开关、错误方块、文件夹、插头、工具箱。

### Content creation

- 选题球、草稿纸、内容包、平台窗口、时间轴、相机、麦克风、评论盒、数据刻度。

### Systems and workflows

- 轨道、接口、分流器、检查门、模板框、档案抽屉、回收环、控制杆、校准尺。

### Risk and validation

- 刹车杆、护栏、锁、盾牌、沙盒、确认按钮、对照尺、放大镜。

## Recurring visual roles

- 黄色耳机：AI 接入、方法提示、重要反馈。
- 钟表尾巴：时间、热点窗口、截止日期。
- 弹簧腿：执行速度、压力和恢复能力。
- 横眼窗：判断、观察和情绪。

## Freshness rule

案例只校准“如何把观点变成动作”。遇到新文章时重新选择动作和物件，不复制案例里的具体布局。

小K可以缩小、倾倒、卡住、被压或在同一画面出现 2-3 个连续状态，但六个身份锚点必须保持可识别。不要把所有场景都画成正面站立全身像。
