# 小K Illustration Skill

## Goal

把内容中的一个观点、动作、冲突或结果，转成清楚、可识别、适合发布的小K角色视觉。小K必须参与核心动作，不能只是站在角落充当装饰。

## Load references

- 每次生成前读取 `references/character-dna.md`，并引用 `assets/ip/k/character-reference.png`。
- 生成情绪、动作组或短漫画时，同时引用 `assets/ip/k/expression-sheet.png`。
- 根据发布渠道读取 `references/platform-formats.md`。
- 同时为公众号和小红书制作发布包时，读取 `references/wechat-xhs-workflow.md`。
- 规划文章或多图内容时读取 `references/content-to-scene.md` 和 `references/composition-patterns.md`。
- 编写生图提示词时读取 `references/prompt-template.md`。
- 交付前读取 `references/qa-checklist.md`。

只加载当前任务需要的参考文件。

## Workflow

1. 识别发布渠道、内容主题、期望数量和是否需要文字。
2. 提取最值得视觉化的观点、动作、冲突和结果，不平均分配插图。
3. 为每张图确定一个核心意思、一个 K 的动作和最多两个主要物件。
4. 先列简短 shot config；用户明确要求生成时，直接调用图片生成工具完成图片。
5. 每张图单独生成。角色母版只约束 K 的身份，不照搬母版动作和构图。
6. 按角色一致性、内容可读性、平台安全区和原创边界验收；失败时只修出错层。
7. 用户未指定目录时在对话中交付，不擅自发布、同步或上传 GitHub。

## Shot config

每张图只记录：

- 渠道与比例
- 核心意思
- K 的动作和情绪
- 主要物件
- 构图模式
- 必要文字及安全区

## Defaults

- 公众号正文和博客横图：`16:9`。
- 公众号封面：`2.35:1`，主体避开右侧裁切区。
- 小红书图文：`3:4`，封面标题不超过两行。
- 抖音、视频号和 Vlog 竖屏素材：`9:16`，主体和文字避开上下及右侧交互区。
- 角色设定、头像和表情：`1:1`。
- 默认暖白背景、粗糙黑色手绘线、少量黄色点缀；无必要不加文字。
- 同一主题需要跨平台时，共享观点和角色动作，但分别重构版式，不直接拉伸或裁切同一张图。

## Boundaries

- 不临摹、换色或微调第三方角色。
- 不要求模仿在世艺术家或特定品牌画风。
- 不复制参考图的具体构图、动作组合和文字设计。
- 不擅自改变 K 的五个身份锚点。
- 不默认添加 Logo、品牌名或 `K` 字母。
- 不把文章配图画成信息拥挤的 PPT。
- 不声称已经完成商标、版权或法律审查。

## Delivery

说明生成数量、用途、比例和保存路径。发现角色漂移、错字、遮挡或安全区问题时，先修正再交付。
